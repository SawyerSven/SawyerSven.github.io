# 为什么<script>标签需要放在<body>最后?

如果JS引擎在修改DOM的同时渲染引擎也在渲染页面，这样会导致渲染引擎获取的DOM元素信息与JS引擎操作DOM后的结果不一致。所以为了避免出现这种情况，GUI线程与JS线程被设计为互斥关系。当JS执行的时候GUI线程会被冻结，GUI的渲染被保存在一个队列里，等待JS引擎空闲时执行渲染。

由于浏览器GUI线程与JS线程互斥，如果将JS放在head中，则会在解析到head的时候调用JS引擎执行javascript中的代码，如果JS正在执行CPU密集型运算，JS线程将会被阻塞，导致渲染线程不能执行渲染，看起来就像页面一直在加载。

所以将JS放在页面尾部，可以保证在页面加载完成后再执行JS代码，这样就不会造成页面加载阻塞。
