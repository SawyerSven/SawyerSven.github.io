# 面试总结

# 标准和模型和IE盒模型有什么区别

(图片来自网络，侵删)

标准盒模型的宽高是content部分的宽高，而不包括padding,border。

IE盒模型的宽高是content+padding+border。

给标准盒模型的元素增加padding和border属性，在审查元素时会发现元素的实际宽高是width/height+padding+border的宽高。

IE模型的元素增加padding和border属性，审查元素时元素宽高为css中设置的宽高，而padding和border挤压了一部分content的内容，所以整个box的宽高并没有改变。

# 谈一谈对Vue生命周期的理解

(图片来自Vue官方文档)

如图所示，Vue生命周期按照先后顺序分为：

  • beforeCreate
  • created
  • beforeMount
  • mounted
  • beforeDestroy
  • Destroyed

以及在数据改变时提供的两个钩子：

  • beforeUpdate
  • updated

另外官方文档的API中还提到了了三个钩子：

  • activated
  • deactivated
  • errorCaptured (2.5.0新增)

接下来详细的了解每个钩子的用处：

  • beforeCreated

在实例初始化之后，数据观测(data observer)和event/watcher事件配置之前被调用；

由于此时数据尚未初始化所以我们无法获取Vue实例的data以及调用methods。

在这个阶段我们通常写入一些loading动画的操作，在created后取消动画。

  • created

在实例创建完成后立即被调用。在这一步，实例已完成以下的配置：数据观测(data observer),属性和方法的运算，watch/event事件回调。然而挂载阶段还没开始，$el属性目前不可见。

在created时，可以获取到实例的属性以及methods.所以在这个阶段我们可以调用method以及获取data的内容。

通常这个阶段我们使用axios获取服务器接口信息。

  • beforeMount

在挂载开始之前被调用：相关的 render 函数首次被调用。

这个阶段由于我们无法获取到refs以及vm.$el，所以我们无法对DOM进行操作。

  • mounted

el被新创建的 vm.$el 替换，并挂载到实例上去之后调用该钩子。如果 root 实例挂载了一个文档内元素，当 mounted 被调用时 vm.$el 也在文档内。

注意 mounted 不会承诺所有的子组件也都一起被挂载。如果你希望等到整个视图都渲染完毕，可以用 vm.$nextTick 替换掉 mounted：

mounted: function () {
  this.$nextTick(function () {
    // Code that will run only after the
    // entire view has been rendered
  })
}

在mounted代表着组件已经挂载完成，在Mounted时我们可以获取到$el以及$refs，所以说可以对DOM进行操作。

在这个阶段我们可以取消beforeCreated的Loading,可以操作DOM的属性。

# 关于在created或mounted进行axios异步获取接口数据。

以下内容均为本人开发习惯，不代表绝对正确。

通常在created里进行axios请求(不包含复杂同步代码)，因为在created的时候就会创建实例，所以在created里可以调用methods里声明的异步函数，并且赋值给data。并且异步请求并不会阻塞render，所以在created里进行异步请求是可以的。

如果在mounted里进行异步请求，因为mounted的触发在实例挂载之后，而实例挂载需要消耗一定的事件，并且当实例挂载完成之后，用户就可以看到页面，如果请求耗时较久，页面没有数据的情况就会越久，所以不在mounted里请求接口。

但是如果请求前需要进行复杂的同步代码,则不推荐在created里进行接口请求，因为在created钩子里执行同步代码会阻塞渲染进程，导致页面必须在同步代码执行完成后才能挂载(可用sleep方法在created时进行测试，会阻塞渲染)。

  • beforeUpdate

数据更新时调用，发生在虚拟 DOM 打补丁之前。这里适合在更新之前访问现有的 DOM，比如手动移除已添加的事件监听器。

  • updated

由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子。

当这个钩子被调用时，组件 DOM 已经更新，所以你现在可以执行依赖于 DOM 的操作。然而在大多数情况下，你应该避免在此期间更改状态。如果要相应状态改变，通常最好使用计算属性或 watcher 取而代之。

注意 updated 不会承诺所有的子组件也都一起被重绘。如果你希望等到整个视图都重绘完毕，可以用 vm.$nextTick 替换掉 updated：

updated: function () {
  this.$nextTick(function () {
    // Code that will run only after the
    // entire view has been re-rendered
  })
}
  • beforeDestroy

实例销毁之前调用。在这一步，实例仍然完全可用。

  • destroyed

Vue 实例销毁后调用。调用后，Vue 实例指示的所有东西都会解绑定，所有的事件监听器会被移除，所有的子实例也会被销毁。

# 前端路由原理

SPA(single page application)是现代前端开发最流行的页面模型。单页面应用指的是应用只有一个主页面，通过动态替换DOM内容并同步修改URL地址，来模拟多页面应用的效果，切换页面的功能直接由前台脚本完成，而不是由后端渲染前端显示。而SPA能够模拟多页面应用的效果，归功于其前端路由机制。

前端路由，顾名思义就是一个前端不同页面的状态管理器，可以不向后台发送请求而直接通过前端技术实现多个页面的效果。angulars中的ui-router,Vue中的vue-router以及react的react-router均是这种功能的具体实现。

# 两种实现方式及其原理

  1. HashChange
  • 原理

HTML页面中通过锚点定位原理可以进行无刷新跳转，触发后URL地址中会多出#+'xxx'部分，同时在全局的window对象上触发hashChange事件，这样就可以在页面锚点哈希改变为某个预设值的时候，通过代码触发对应的页面DOM改变，就可以实现基本的路由了，基于锚点哈希的路由比较直观，也是一般前端路由插件中最常用的方式。

  1. HTML5 HistoryAPI
  • 原理

HTML5的HistoryAPI为浏览器全局history对象增加的扩张方法。一般用来解决ajax请求无法通过back按钮回到请求前状态的问题。

在HTML4中，已经支持window.history对象来控制页面历史记录的跳转，常用方法包括：

  • history.forward(); //在历史记录中前进一步
  • history.back(); //在历史记录中后退一步
  • history.go(n); // 在历史记录中跳转n步骤，n为刷新本页，n = -1为后退一页。

在HTML5中，window.history对象得到了扩展，新增的API包括：

  • history.pushState(data,[title][,url]) //向历史记录中追加一条记录

  • history.replaceState(data[,title][,url]) // 替换当前页在历史记录中的信息

  • history.state; //是一个属性，可以得到当前页的state信息。

  • window.onpopstate; // 是一个事件，在点击浏览器后退按钮或js调用forward()、back()、go()时触发。监听函数中可传入一个event对象，event.state即为通过pushState()或replaceState()方法传入的data参数。

    • 应用

    浏览器访问一个页面时，当前地址的状态信息会被压入历史栈，当调用history.pushState()方法向历史栈中压入一个新的state后，历史栈顶部的指针是指向新的state的。可将其作用简单理解为假装已经修改了url地址并进行了跳转,除非用户点击了浏览器的前进,后退,或是显示调用HTML4中的操作历史栈方法，否则不会触发全局的popstate事件。

    在下面的示例中,点击导航按钮，可以看到url地址栏发生了变化，且控制台打印出了响应的信息。


<body>
    <ul id="list">
        <li>link1</li>
        <li>link2</li>
        <li>link3</li>
    </ul>
</body>
<script>
    window.onload = function () {
        window.onpopstate = function (e) {
            console.log(e);
        }
        window.onhashchange = function (e) {
            console.log('catch HashChange event');
        }
    }

    document.querySelector('#list').addEventListener('click', function (event) {
        if (event.target.nodeName.toLowerCase() == 'li') {
            var content = event.target.innerHTML;
            var _newState;
            _newState = {
                url: location.origin + '/' + content,
                title: document.title,
                state: content
            };
            window.history.pushState(_newState, '', '/' + content);
            console.log(`you have change the router to : ${content}`);
        }
    })

</script>
