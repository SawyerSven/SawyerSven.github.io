# 前言

ECMAScript通俗的理解可以理解成Javascript的一个标准,但是其实Javascript是ECMA-262标准的实现和扩展

# 为啥要看ECMAScript(ECMA-262)

  就我个人来说，由于是个big学渣加上毕业以后英语基本上一年还给老师一点，差不多该忘掉的都忘掉了，该记住的也忘得差不多了，所以内心一向比较抗拒查看英文文档(Vue文档我都是看中文版的)。所以期初对ECMAScript的规范并没有任何想看的欲望。

  但是最近在一个前端群里有个群友问到一些问题的时候，发现自己了解的都是从MDN或者其他一些网站看到的结论性的表述，只了解这个语法会有什么效果，但是不了解这个语法为什么会有这个效果,所以，为了知其然并知其所以然，我决定开始通过ECMA-262规范中对他们的定义去深入了解Javascript语法的一些被我忽略的东西

总而言之言而总之，规范嘛。肯定是大家都一定要去提倡去遵守的，阅读规范有助于我们更深层次的了解Javascript的运行原理，当我们了解了运行原理，下次实习生小妹妹问你问题的时候不就是大展身手的时候我们就可以少写BUG，早点回家，不要秃头。

# 阅读需知

  1. 本文撰写时主要参考ECMA-262-6版本

  2. 本文具有很强烈的主观性，大多都是博主个人的理解。内容都是博主一个人阅读ECMAScript并通过代码进行测试后得出的结果。不保证结果的正确性，建议结合ECMA-262和其他相关文献进行对比阅读，以免由于博主的才疏学浅影响更多人对某些知识点的理解。

  3. 本文的内容主要是博主个人学习并加以总结，行文思路会比较偏向本人的薄弱的知识点，所以，如果有部分内容您感觉简单或者已经掌握了，可以跳过。

  4. 🎉再强调一次吧，真的真的可能会有很多博主理解不到位的内容，如果您发现文章中有错误支出，还望不吝指出，共同学习，共同进步。github

  5. 本文会翻译部分ECMA-262的内容方便行文，但本文不等同于ECMA-262的译文。如果需要精确的翻译，请自行搜索相应翻译文档或自行查看原文档。
